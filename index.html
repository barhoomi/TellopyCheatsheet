DJITelloPy Cheat Sheet

A quick reference for the Ryze Tello drone library.

Connection & Setup

tello = Tello()Creates a Tello instance.
tello.connect()Enters SDK mode. Must be called first.
tello.end()Closes connection. Call at the end.
tello.reboot()Reboots the drone.

Flight & Emergency

tello.takeoff()Auto takeoff.
tello.land()Auto landing.
tello.emergency()Stops all motors immediately.
tello.initiate_throw_takeoff()Enables throw-to-takeoff.

Flip Commands

tello.flip_left()Flip to the left.
tello.flip_right()Flip to the right.
tello.flip_forward()Flip forward.
tello.flip_back()Flip backward.

Basic Usage Example

from djitellopy import Tello
import time

tello = Tello()

tello.connect()
print(f"Battery: {tello.get_battery()}%")

tello.takeoff()
tello.move_forward(50)
time.sleep(2) # Wait for the movement to complete
tello.land()
tello.end()

Tello & Computer Vision Example (Live Stream)

This example demonstrates how to capture live video frames from the Tello drone and display them using OpenCV. Remember to have OpenCV installed (pip install opencv-python).

import cv2
from djitellopy import Tello
import time

tello = Tello()
tello.connect()
tello.streamon() # Start the video stream

frame_read = tello.get_frame_read()

while True:
    # Read a frame from the stream
    img = frame_read.frame

    # Perform a simple image processing task (e.g., convert to grayscale)
    gray_img = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    # Display the original and processed frames
    cv2.imshow("Tello Original Frame", img)
    cv2.imshow("Tello Grayscale Frame", gray_img)

    # Break the loop if 'q' is pressed
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

# Clean up
tello.streamoff() # Stop the video stream
tello.end()
cv2.destroyAllWindows()

Training a YOLO Model Example (Conceptual)

This is a conceptual example demonstrating the typical steps involved in training a YOLO (You Only Look Once) object detection model, often using a library like Ultralytics YOLO. Actual training requires a properly prepared dataset, configuration files, and significant computational resources. Data for training could potentially be collected by drones.

# Example using Ultralytics YOLOv8 (install with: pip install ultralytics)
from ultralytics import YOLO

# 1. Load a pre-trained YOLOv8n model (nano version, good starting point)
#    You can also specify a custom .yaml file if training from scratch
model = YOLO('yolov8n.pt') 

# 2. Define the path to your dataset configuration file
#    This YAML file specifies paths to your images, labels, and class names
data_config_path = 'path/to/your/dataset.yaml' 

# 3. Train the model
#    'data': path to your dataset config
#    'epochs': number of training epochs
#    'imgsz': image size for training
#    'batch': batch size
#    'name': name for the training run results
results = model.train(data=data_config_path, 
                      epochs=100, 
                      imgsz=640, 
                      batch=16,
                      name='custom_yolo_training_run')

# 4. (Optional) Validate the trained model
metrics = model.val() 

# 5. (Optional) Export the model to a different format (e.g., ONNX, OpenVINO)
#    This is useful for deployment
model.export(format='onnx') 

print("\nYOLO model training conceptual steps completed.")
print("Remember: Real-world training requires data labeling, environment setup, and tuning.")
